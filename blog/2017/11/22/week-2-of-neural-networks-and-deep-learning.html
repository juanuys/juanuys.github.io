Binary Classification

Video #1 basically sets up some notation for showing how a matrix input maps to one bit of information (1 or 0, true or false, i.e. whether a picture is of a hotdog or not).

Logistic regression

Video #2 shows how Logistic Regression trumps Linear Regression in the sense that the latter will output values in the range negative to very large (>1), which is not what we want for binary classification. Instead, Logistic regression ‚Äúclamps‚Äù the output to 0 <= y^ <= 1 using a sigmoid function. ‚Äúy hat‚Äù, i.e. y^ will be the prediction of y, the ‚Äúground truth‚Äù.

Logistic regression cost function

Video #3 explains the cost function needed to train parameters w and b. Square error doesn‚Äôt work well as a loss (or error) function, because no global minimum can be obtained. It makes gradient descent not work well at all (your ‚Äúball‚Äù gets trapped in the first ‚Äúhole‚Äù as it rolls down the hill). We want a function which ends us up with one convex basin, not a squiggly line (many basins).

The cost function measures how well you‚Äôre doing over the entire training set.

Gradient descent

Video #4 discusses gradient descent as an ‚Äúoptimisation algorithm‚Äù shows how logistic regression can be used as a tiny neural net.

Gradient descent is your ‚Äúball rolling to the bottom of the basin‚Äù, or final a local optimum for w. It shows how taking the derivative (or the slope) of the curve finds the optimum where the slope is 0 (a flat line underneath the curve). For the multi-dimensional case where we‚Äôre finding w AND b, it would be the plane under the basin.

The latter part of the video makes a brief note of calculus where you can use d for the derivative of a single-variable function, or squiggly-d as the partial derivative of a multi-variable function, or the cost function, in our case: J(w,b)

Vectorising logistic regression‚Äôs gradient computation

Video #13 explains logistic regression for m samples using numpy.

It has Z = np.dot(w.T, x) + b from before, but adds

A = sigmoid(Z)
dZ = A - Y
dw = 1/m * X * dZ.T
db = 1/m * np.sum(dZ)
w := w - sigmoid(dw)
b := b - sigmoid(db)

Then he wraps it with a range(1000) outer loop, and I guess it will become more apparent later that these extra iterations gets your weights to a value where they ‚Äúsettle‚Äù (and you can probably break out of the loop if the delta becomes neglible).

What else I‚Äôve learned

Clean and pre-process

Before building a neural net, you‚Äôll need to pre-process your data. (This is the same gist I picked up when doing my NN diploma back in 2001: cleaning and preparing data is sometimes the hardest part)

Common steps for pre-processing a new dataset are:

  • Figure out the dimensions and shapes of the problem (m_train, m_test, num_px, ‚Ä¶)
  • Reshape the datasets such that each example is now a vector of size (num_px * num_px * 3, 1)
  • ‚ÄúStandardize‚Äù the data, e.g. divide each pixel by 255

The oval ‚Äúneuron‚Äù in this diagram computes a linearity (Wx + b) and then an activation (sigmoid, tanh, ReLU) which is the prediction.

Key steps to building a model

  • define the model structure, such as number of input features
  • initialize the parameters of the model, e.g. initialise the weights w to zeros or ones, or random numbers, and initialise the bias b
  • loop
    • forward propagation: calculate current loss; from X to cost
    • backward propagation: calculate current gradient
    • update parameters (weights and bias): this is gradient descent!

You have a choice of activations

  • sigmoid
  • tanh
  • ReLU

You have a choice of loss functions:

  • L1, or ‚Äúleast absolute deviations‚Äù (LAD), or ‚Äúleast absolute errors‚Äù (LAE)
  • L2, or ‚Äúleast squares error‚Äù (LSE)
  • L3? (as per week2) or ‚Äúlog-likelihood‚Äù?

Good activations and loss functions seem to be ‚Äústumbled upon‚Äù empirically, and must be a topic of research.

Picking good hyperparameters (number of iterations, and learning rate) is an art? Different learning rates give different costs and thus different predictions results.

An example of good/bad learning rates are illustrated below: good rate converges, and a bad rate diverges. (Images: Adam Harley)

If the learning rate is too large, the cost may oscillate up and down, or diverge completely. Lower cost isn‚Äôt necessarily better, and could lead to overfitting. It happens when the training accuracy is a lot higher than the test accuracy.

More techniques in later weeks to reduce overfitting.
